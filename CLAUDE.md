## 專案概述

### 專案名稱
**尋找你的「樹」適圈** — 台北市路樹風險查詢平台

### 課程背景
政治大學「社會傳播導向大數據分析」課程期末專案。以**融媒體報導**形式呈現，結合深度報導與互動查詢平台，讓讀者在閱讀報導的同時可即時查詢自身周遭路樹的風險狀況。

### 核心議題
台北市路樹倒塌事件背後的**制度性失靈**：
- 褐根病是路樹倒塌的主要原因（約佔 60%），但外觀難以辨識
- 台灣缺乏專業樹藝師制度（全台僅約 90 位 ISA 認證樹藝師）
- 22 縣市中只有 7 個建立褐根病防治措施
- 逢甲大學學生被倒塌榕樹壓死案（2024），檢方因無法證明管理疏失不起訴

### 呈現形式
1. **深度報導網頁**（另一位組員負責）
2. **「樹」適圈互動查詢平台**（本工作負責）— 讓使用者輸入地址或在地圖上點選，查詢附近路樹的風險等級與健康狀態

## 重要發現與數據（報導核心）

### 風險分析
- 台北市共 **4,427 棵高風險路樹**（自建風險評分：高風險樹種 + 胸徑 > 50cm + 靠近列管老樹）
- 官方認定**高/重風險路樹：48 棵**（health_風險結果 = 高 或 重）
- **士林區高風險比例最高（8.1%）**，內湖區最低（1.8%）
- 高風險樹種（榕樹、黑板樹）佔全市行道樹 **16.3%**

### 褐根病
- 台北市 2019-2026 年共通報 **5,107 件**病害案例
- 其中**褐根病 2,935 件**（57.5%）
- 2023 年通報最高峰（574 件）

### 制度面
- 褐根病防治：22 縣市中只有 **5 個**建立
- 全台 ISA 認證樹藝師約 **90 位**，攀樹師只有 **4 位**
- 台北市 108-112 年路樹國賠 **48 件，賠償 952 萬元**，居全國之冠

## 查詢平台功能需求

### 核心功能
1. **地圖顯示**：台北市所有行道樹位置，依風險等級顯示顏色
   - 高/重風險：紅色
   - 中風險：橘色
   - 低/D級：綠色
   - 無資料：灰色

2. **搜尋功能**：輸入地址或路名，找出附近的高風險路樹

3. **樹木詳細資訊**：點擊地圖上的樹，顯示：
   - 樹種、胸徑、樹高
   - 官方風險等級
   - 健檢扣分
   - 健檢關鍵因子（問題描述）
   - 行政區、道路位置

4. **行政區統計面板**：顯示各行政區高風險比例

### 技術規格
- 前端框架：Vue.js 或 React（依組員選擇）
- 地圖套件：Leaflet.js 或 Mapbox
- 資料來源：`tree_integrated.csv`（本機讀取或轉成 JSON）
- 座標系統：`tree_integrated.csv` 需另外提供 WGS84 經緯度版本

### 注意事項
- `tree_integrated.csv` 的座標是 TWD97，需轉換成 WGS84（經緯度）才能用在 Leaflet
- 轉換方法：在 R 裡用 `sf` 套件轉換，或在 JS 裡用 `proj4` 套件
- 資料量大（92,927 筆），建議前端做分區載入或使用 tile server

## 重要注意事項

### 風險評分說明
本專案的「風險評分」是**自行建立的分析框架**，非官方標準：
- 高風險樹種（+1分）：榕樹、菩提樹、盾柱木、黑板樹
- 大直徑老樹（+1分）：胸徑 > 50 公分
- 靠近列管老樹（+1分）：10 公尺內有受保護樹木

官方風險等級來自 geopkl（`health_風險結果` 欄位），這才是可以對外引用的數據。

### 資料配對說明
`tree_integrated.csv` 的健康資料是用 **20 公尺空間距離配對**，松山區和信義區因 geopkl 資料覆蓋不完整，未配對比例較高。在平台說明中應標注哪些樹木「尚無健檢資料」。

### 版權與引用
- 台北市行道樹資料：政府資料開放授權條款，可自由使用
- geopkl 健康資料：台北市政府工務局公園路燈工程管理處
- 林試所褐根病資料：林業試驗所，引用需標注來源
- 監察院報告數據：公開資料，引用需標注出處
